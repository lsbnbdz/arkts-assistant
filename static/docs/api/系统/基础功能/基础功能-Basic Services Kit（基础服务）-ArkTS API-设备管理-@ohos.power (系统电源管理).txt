该模块主要提供重启、关机、查询屏幕状态等接口。开发者可以使用该模块的接口获取设备的活动状态、电源模式、亮灭屏状态等。

 说明 本模块首批接口从API version 7开始支持。后续版本的新增接口，采用上角标单独标记接口的起始版本。

 
## 导入模块
PhonePC/2in1TabletWearable收起自动换行深色代码主题复制
```
import {power} from '@kit.BasicServicesKit';
```
 
## power.isActive9+
PhonePC/2in1TabletWearableisActive(): boolean

 检测当前设备是否处于活动状态。

 - 有屏的设备亮屏时为活动状态，熄屏时为非活动状态。
- 无屏的设备非休眠时为活动状态，休眠时为非活动状态。

 **系统能力：** SystemCapability.PowerManager.PowerManager.Core

 **返回值：**

 展开
| |
| |
| |
| boolean | 活动状态返回true，非活动状态返回false。 |

 **示例：**

 收起自动换行深色代码主题复制
```
let isActive = power.isActive();console.info('power is active: ' + isActive);
```

## power.isScreenOn(deprecated)
PhonePC/2in1TabletWearableisScreenOn(callback: AsyncCallback<boolean>): void

 说明 从API version 7开始支持，从API version 9开始不再维护，建议使用[power.isActive](https://developer.huawei.com/consumer/cn/doc/harmonyos-references/js-apis-power#powerisactive9)替代。

 检测当前设备的亮灭屏状态。使用callback异步回调。

 **系统能力：** SystemCapability.PowerManager.PowerManager.Core

 **参数：**

 展开
| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| callback | AsyncCallback | 是 | 回调函数。当检测成功，err为undefined，data为获取到的亮灭屏状态，返回true表示亮屏，返回false表示灭屏；否则为错误对象。 |

 **示例：**

 收起自动换行深色代码主题复制
```
power.isScreenOn((err: Error, data: boolean) => { if (typeof err === 'undefined') { console.info('screen on status is ' + data); } else { console.error('check screen status failed, err: ' + err); }})
```
